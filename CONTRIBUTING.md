# Contributing

Thank you for wanting to contribute! Please follow these steps to make the process smooth for everyone.

1. Read the Community Standards Checklist
   - See .github/COMMUNITY_STANDARD_CHECKLIST.md for expectations and required checks before filing issues or PRs.

2. Search existing issues
   - Look for duplicates before opening a new issue or feature request.

3. Issue templates
   - Use the provided issue templates when filing bugs or feature requests.

4. Branching & commits
   - Create a descriptive branch: feature/..., fix/..., docs/...
   - Write clear commit messages and keep changes focused.

5. Tests & CI
   - Add or update tests for any code changes and ensure CI passes.

6. PR description
   - Explain the problem and your solution.
   - Reference related issues (e.g., fixes #123).
   - Include the checklist from .github/COMMUNITY_STANDARD_CHECKLIST.md in the PR body or confirm it has been followed.

7. Code style
   - Follow the project's formatting and style (e.g., black/isort for Python, if applicable).

Maintainers will review PRs and provide feedback. If you need assistance, contact the maintainers at sany2k8@gmail.com.
